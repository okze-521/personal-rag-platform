# 核心组件环境部署实施 (Deployment Steps)

**章节目标**: 提供节点一/节点二的 Docker Compose 及环境变量配置参数。

## 1. 节点一：逻辑控制层 (Laptop / Localhost)

### 1.1 Qdrant 向量库部署
在 `/configs/docker-compose-qdrant.yaml` 中定义持久化挂载策略：

```yaml
services:
  qdrant-vector-db:
    image: qdrant/qdrant:v1.9.0
    volumes:
      - ../data/qdrant_storage:/qdrant/storage   # 绑定到 data/ 盘确保重启不丢失
      - ./config.yaml:/qdrant/config/production.yaml  # 挂载自定义配置
    ports:
      - "6333:6333"          # REST API
      - "6334:6334"          # gRPC (Docker internal)

  # Ollama for Embedding
  ollama-embedder:
    image: ollama/ollama:latest
    volumes:
      - ../data/ollama_models:/root/.ollama/models
    environment:
      - OLLAMA_HOST=0.0.0.0:11435  # 笔记本独立端口防冲突
```

### 1.2 BGE-M3 / RAG Demo 配置
*   **路由文件**: `src/configs/endpoint.env`
*   **关键变量定义**:
    ```bash
    VECTOR_STORE_URL="http://127.0.0.1:6333"
    EMBEDDING_MODEL="bge-m3:latest"
    # 指向远程台式机算力节点 (MoE)
    INFERENCE_API_BASE="http://192.168.3.200:11434"
    ```

## 2. 节点二：算力推演层 (Desktop / 192.168.3.200)

### 2.1 Ollama 服务开启与局域网放行
Windows 端修改 `.ollama` 环境变量，允许笔记本访问：

*   **路径**: `C:\Users\okze\.ollama\envvars` (或系统环境变量)
*   **配置内容**:
    ```bash
    OLLAMA_HOST=0.0.0.0:11434     # ⚠️ 必须改为 0.0.0.0，否则仅监听 localhost
    OLLAMA_DEBUG=1                  # 开启日志便于排查跨网段请求失败问题
    ```

### 2.2 Qwen3.6 (MoE) 模型拉取与大文件缓存
针对台式机硬件环境的调优参数：

```bash
# 拉取高精度 MoE 量化版
ollama pull qwen3.6:35b-a3b-q4_K_M

# 设置显存分配策略 (Windows Ollama App.exe 高级设置中)
- GPU Layer: -1  (表示自动探测并全量加载至显卡)
- Context Size: 8192 (按需调整，防止显存溢出)
```

## 3. 部署验证方法

| 步骤 | 操作指令 / 测试点 | 预期结果 |
|------|-------------------|----------|
| **Step 1** | `docker ps` (笔记本端) | Qdrant (`:6333`) 与 Ollama Embedder (`:11435`) 状态正常 |
| **Step 2** | 远程访问 `http://192.168.3.200:11434` (笔记本端) | 返回 HTTP 200 或 JSON tags 列表 |
| **Step 3** | `ollama run qwen3.6:35b-a3b-q4_K_M "hello"` | MoE 激活成功，生成正常响应流输出 |
