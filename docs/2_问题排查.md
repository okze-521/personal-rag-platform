## 🔧 维护操作规范 (Maintenance Procedures)

### 步骤一：模型更新/切换
1. **笔记本端**: `ollama pull bge-m3:latest` — 拉取最新 Embedding 权重。
2. **台式机端**: 
   * `ollama rm qwen3.6:35b-a3b-q4_K_M` — 删除旧模型缓存。
   * `ollama pull qwen3.6:35b-a3b-q4_K_M` — 拉取 MoE 量化版。

### 🔧 **步骤二：RAG Demo 服务重启**
1. 停止调度脚本 (`Ctrl + C`)。
2. 清理 Docker 临时缓存: `docker compose down --volumes` (⚠️仅开发环境)。
3. 启动日志监控: `docker logs -f ollama-embedder qdrant-vector-db`

## ⚠️ **运维安全规范 (Security Protocol)**

1. **端口暴露原则**: 除笔记本本机 (`127.0.0.1:6333`) 外，禁止将 Qdrant/Ollama 任何端口暴露到公网。Z4 Pro / 路由器需严格配置防火墙规则。
2. **网络隔离建议**: RAG 计算层 (台式机) 建议使用独立 VLAN 或与日常办公网段物理/逻辑隔离，防止中间人攻击窃取 MoE 推理结果。
3. **敏感数据脱敏**: 在生产级 RAG Demo 路由中，需在 `rag_demo.py` 输入层加入关键词过滤与敏感字段加密（PiiFilter / RegexMask）。
